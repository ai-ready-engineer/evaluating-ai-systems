LLM as a Judge

L2 lab · an LLM can be the thing that judges your AI's answers — classifying them or scoring them. In L1 the judge was perfect; a real judge has two defects, bias and noise.

Uncertainty dimension: judge noise & bias.

Where L1 left off

Lesson 1 made two quiet assumptions: there is a perfect ground truth, and we can compare the AI's answer to it perfectly. The only uncertainty was which cases landed in your test set (sampling noise). This lesson removes the second assumption. The thing that judges an answer is itself an instrument, and instruments are imperfect in two distinct ways.

Bias — systematic

The judge is consistently off in one direction — a classifier that keeps reaching for the same wrong label, or a scorer that rates everything a few points high.

Effect: your measurement is shifted off the truth. More data does not fix it — it just makes you confidently wrong.

Noise — random

The judge is inconsistent: re-run it (temperature, an ambiguous rubric, a different annotator) and the same answer gets a different label, or a different score.

Effect: your measurement wobbles. It averages out with more judgments — but on a finite test set it widens the spread you must report.

We start with the simplest LLM judge — one that classifies — then add the two defects, then move to judges that score. Pick a tab:

An LLM as a classifier

An LLM can classify, exactly like the trained models in L1 — read a piece of text and emit one label (POSITIVE / NEGATIVE, or one of several categories). No training: a prompt with the allowed classes and a few labelled examples. This demo moved here from the L1 lab — same datasets, real precomputed LLM predictions; spin freely, no API calls.

Because we hold the true labels, we can measure the LLM exactly like L1 measured the trained model: accuracy, per-class precision/recall, the confusion matrix. And the confusion matrix shows the direction of the misses — the bias — visible only because we have ground truth.

The catch: how do you know the judge's labels are right?

      

        The first tab could measure the LLM judge only because the L1 datasets come with gold labels.
        In real judging work that's the exception, and everything depends on which side of it you're on.
      


      

        

          Ground truth available
          
You hold trusted labels. You can measure the judge directly — its accuracy, and crucially its
             bias: does it systematically lean toward one label?

          
This is the first tab's setting — gold labels exist, so every judge error is visible.

        

        

          No ground truth
          
Fresh data nobody labeled. You can run the judge, but you can't tell if a label is right — only
             whether the judge is consistent on re-runs (that's its noise).

          
To see or fix bias you must calibrate against a small
             trusted slice. With zero labels, bias is invisible.

"Calibration" means two different things

      
When there's no ground truth, the fix is calibration — but that word points at two genuinely different
         procedures depending on whether you're calibrating a classic ML classifier or an LLM judge.


      

        

          ML classifier → probability calibration
          
Most ML classifiers output a probability per class. "Calibrated" means the probabilities
             match reality: of the cases it calls 80% POSITIVE, about 80% really are. You check it with a
             reliability diagram and ECE, and fix it with Platt / isotonic / temperature
             scaling.

        

        

          LLM judge → tune the judge
          
You rarely get trustworthy per-class probabilities from an LLM (logprobs aren't always exposed, and a
             verbalized "I'm 90% sure" is badly miscalibrated). So "calibrating" an LLM judge usually means something
             else: tuning it to agree with a trusted slice — mostly by working the
             prompt / rubric (sharper criteria, few-shot anchor examples), plus model choice and
             self-consistency. You still need a small labeled set to tune against.

Bootstrap: how sure are you about that number?

      

        You ran the judge once and got a single mean score. How much would it move if your test set had been a different
        draw, and the noisy judge had rated differently? Bootstrap answers that without any formula:
        resample the test set with replacement thousands of times, re-score each resample, and look at the spread
        of the results. (It's the model-free cousin of L1's Beta interval, and it works for any score — binary, 1–5, or
        continuous.)

        The asymmetry that matters. Raise noise → the bootstrap
        distribution gets wider: noise is honest, it shows up as a bigger interval you can report and then shrink
        (average more judgments, tighten the rubric, drop the temperature). Slide
        bias → the whole distribution shifts away from the green truth line,
        and the interval can sit entirely in the wrong place. The bootstrap only resamples what the judge said, so
        it cannot see the bias — it hands you a tight, confident interval around the wrong answer. Now push
        test-set size up with bias on: the interval gets narrower and excludes the truth even more
        decisively. More data cures noise; it cannot cure bias.
      

    

  


  
  

    
The L2 takeaway

    
    
      
  1. Bias and noise are different defects. Noise is random and averages out; bias is systematic and
        persists. Telling them apart is the first move.
    2. 
      
  2. Both widen your real uncertainty — differently. Noise widens the interval you can measure.
        Bias adds an offset your interval can't even see, so the truth may lie outside a confident-looking CI.
    3. 
      
  3. They call for different fixes. Reduce noise by averaging repeats, lowering temperature,
        and writing a crisper rubric. Reduce bias only by auditing the judge against trusted labels — calibrating
        a "judge of judges" (the thread we pick up in L6).
    4. 
      
  4. Label or number, the same two defects apply. And with no ground truth you can only see
        noise (re-run consistency) — measuring bias always needs a trusted slice to calibrate against.
