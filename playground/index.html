AI Evaluation Playgrounds

One hands-on lab per lesson
Interactive labs for AI-Ready Engineers, organized to follow the eight lessons. Each lesson has one headline playground that lets you feel its source of uncertainty firsthand, then shows you how to reduce it. Below the lessons are the foundations (general stats primers and extra source-of-uncertainty labs) and the Python notebooks that run live LLM calls.
The Eight Lessons
Lesson 1 “Eval” is now an Experiment

Basic classification + noise

A trained ML classifier (TF-IDF) over real datasets. Set a test-set size, spin the wheel, and watch accuracy, the confusion matrix, and a Beta credible interval move. A score is one draw. (The gen-AI classifier moved to the L2 lab.)

interactive sampling noise Lesson 2 Assessing one “run” of an Agent · Part I

LLM as a Judge

The LLM as the thing that judges your AI's answers. Start with the simple case — an LLM that classifies, straight from L1 (real predictions, confusion matrix, the lean that is bias) — then lose the ground truth and see what “calibration” becomes. Then the scoring judge: turn the bias and noise dials, bootstrap, and watch noise widen the interval while bias slides it off the truth.

interactive judge noise & bias Lesson 3 Proportions, Probabilities, Beliefs

Bias–Variance–Noise Explorer

Pull apart the three things that move a measurement — a systematic offset, spread, and random noise. Adjust the distributions and watch the shape, the mean, and the uncertainty change.

interactive distributions Lesson 4 Closed-Book Overfitting

Lucky-Winner Finder

You test 10 prompt variants. The winner scores 4.3 — ship it? All variants had the same true score; the “winner” was just the luckiest. See how selection bias grows with the number of things you try.

interactive selection bias Lesson 5 Open-Book Overfitting

Holdout Simulator

Iterate on a prompt against a dev set and watch the score climb across versions — then check the held-out set and see the gap. The difference between fitting the data and fitting the world.

interactive process discipline Lesson 6 Subjectivity & Noise in Judges and Ground Truth

Judge Auditor

Audit an LLM judge against trusted human labels: measure agreement, surface systematic bias, and see how a subjective rubric shifts the score — before you trust the judge to grade anything.

interactive judge of judges Lesson 7 Additional sources of error — and why they compound

Temporal Drift Lab

Your January eval says nothing about April. Simulate 12 months of gradual degradation and sudden shocks, and see how long it takes to notice without continuous monitoring. Pairs with the compounding visualizer below.

interactive fresh data over time Lesson 8 Designing Reliable Experiments

Submit an eval write-up and check whether the evidence supports the decision. The capstone tool that turns a measured result into a ship, canary, or iterate recommendation.

Foundations & Extra Tools
General-purpose stats primers and the extra source-of-uncertainty labs. Use them alongside any lesson.

N Coin Flips

Watch the bell curve emerge as you add coin flips — a visual introduction to the Central Limit Theorem and the convolution of independent random variables.

CLT interactive

Bootstrap Lab

Hands-on bootstrapping: resample your data, build sampling distributions, compute confidence intervals, and test whether two samples differ. The engine behind many of the uncertainty fixes.

bootstrap interactive

Model Accuracy Estimator

Enter successes out of n trials and get a posterior over accuracy: point estimate, mode, and 95% credible interval. Powered by the Beta distribution with selectable priors. (The L1 uncertainty calculator.)

interactive beta posterior

Sampling Noise

Run the same evaluation twice — get different numbers. Set true scores, pick a sample size, and watch the scorecard wobble. Then tame it with beta posteriors, bootstrap CIs, and confidence intervals.

interactive more data helps

Sampling Bias

Your test set is 70% English but production is 40%. The scorecard looks great — until you see what users actually experience. Explore biased vs. representative sampling with adjustable mixes.

interactive more data doesn’t help

Variance Across Domains

Aggregate says 4.0, but Client A sees 4.5 and Client B sees 3.0 — the aggregate hides the story. Includes a Simpson’s Paradox demo where the “winning” system actually loses everywhere.

interactive property of the world

Choice of Metric

Mean says A wins. Threshold rate says B wins. Worst-case says B wins clearly. The winner changes with every tab click — the data didn’t change, the question did.

interactive design decision

LLM Evolution

GPT-4: 97.6% → 2.4% in 3 months. Same prompt, same task — the model changed under you. Real documented cases plus a simulated version timeline.
