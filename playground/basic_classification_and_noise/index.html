Basic classification + noise

L1 lab · multi-label text classification with two classifiers, three demos.

Uncertainty dimension: sampling noise — a score is one draw.

What this lab does

We pick a multi-label text-classification task — labelling support tickets with tags like billing, urgent, refund — and run two classifiers on a subsample of the data:

  • TF-IDF + Logistic Regression, one-vs-rest. Fast, deterministic, no LLM.
  • LLM few-shot. A small prompt with 2–3 labelled examples; model and key from .env.

We then re-sample many times. The headline accuracy wobbles. Rare labels wobble a lot. A trivial "predict the common tags" baseline catches up on aggregate metrics as the data skews — and the gap on macro-F1 is the only honest signal.

The single thing to feel: a single F1 number is one draw from a distribution. The number you report could have come out 5 points higher or lower if the test set had been sampled differently — and rare labels are noisier still.

Per-label spread is wider still — common labels like joy wobble by ~3 points, rare labels like grief wobble by 30+ because a single case being misclassified moves the score wildly.

What to notice: with the same data, same classifier, the headline number swings by ~10 points across resamples. If you ran your eval once and reported "0.59", you'd be calling something a regression that another draw would have shown as "0.68".

What to notice: as the data skews more, the trivial baseline catches up to the real classifier on micro_f1 (which is dominated by common labels). On macro_f1 — which weights every label equally — the gap stays wide. The micro number flatters the trivial baseline; the macro number is the only honest comparison.

Running it yourself

Two paths:

  • Code path: open notebook.ipynb. pip install scikit-learn datasets pandas python-dotenv jinja2 matplotlib, then run the cells top to bottom. To run the LLM live, copy .env.example to .env and fill in an API key; otherwise a deterministic mock runs and the metric flow still works.
  • Inspect path: read lab.py for the dataset loaders, the TF-IDF+LogReg classifier, the few-shot LLM wrapper, and the evaluator. Read prompts/multilabel_classify.j2 for the prompt.

First-time setup: run python ../../datasets/cache_datasets.py to cache Bitext and GoEmotions from HuggingFace. The synthetic ITSM dataset is already committed; regenerate at different skew levels with python ../../datasets/synthetic_itsm/generate.py --skew heavy.
