Basic classification + noise

L1 lab · evaluate two classifiers — a trained TF-IDF model and a gen-AI few-shot model — on three datasets.

Uncertainty dimension: sampling noise — a score is one draw.

What this is

We have three text-classification datasets and, for each, two fixed classifiers: a trained TF-IDF + Logistic Regression model and a gen-AI few-shot LLM. Predictions and ground truth are precomputed — the page just resamples them, so you can spin freely with no waiting and no API calls.

Two questions drive the lab: (1) given a classifier, how do we decide it's good — which metrics, and good enough for what? (2) How much should we trust a single score? Every number here is one draw from a test set; spin the wheel and watch it move, then read it honestly with the uncertainty tool.

Metrics are computed per property (the thing being classified): for the single-label category dataset that's one 5-value confusion matrix; for the multi-label datasets each label is its own yes/no property with its own accuracy, confusion matrix, and precision/recall.

Running it yourself

The numbers come from predictions/<dataset>.json, produced by build_predictions.py (trains TF-IDF once, runs the LLM in batches, caches everything). Regenerate with a key in .env (LIVE=true, OPENROUTER_API_KEY) via python build_predictions.py. The code path is notebook.ipynb.
